## <a name="configuring-web-server-permissions"></a><span data-ttu-id="a4238-119">Konfigurieren von Webserver Berechtigungen</span><span class="sxs-lookup"><span data-stu-id="a4238-119">Configuring Web Server Permissions</span></span>

<span data-ttu-id="a4238-120">Wie unter [auswählen des richtigen Ansatzes für die Webbereitstellung](../configuring-server-environments-for-web-deployment/choosing-the-right-approach-to-web-deployment.md)beschrieben, gibt es zwei Hauptansätze, die Sie verwenden können, wenn Sie Webpakete auf einem Remoteweb Server bereitstellen möchten:</span><span class="sxs-lookup"><span data-stu-id="a4238-120">As described in [Choosing the Right Approach to Web Deployment](../configuring-server-environments-for-web-deployment/choosing-the-right-approach-to-web-deployment.md), there are two main approaches you can use if you want to deploy web packages to a remote web server:</span></span>

- <span data-ttu-id="a4238-121">Stellen Sie die Anwendung von einem Remote Speicherort aus bereit, indem Sie auf dem Zielserver den *Web Deployment Agent-Dienst* (auch als Remote-Agent bezeichnet) Anzielen.</span><span class="sxs-lookup"><span data-stu-id="a4238-121">Deploy the application from a remote location by targeting the *Web Deployment Agent Service* (also known as the remote agent) on the destination server.</span></span>
- <span data-ttu-id="a4238-122">Stellen Sie die Anwendung von einem Remote Speicherort aus bereit, indem Sie auf den *Internetinformationsdienste* (*IIS) Web deploy Handler* auf dem Zielserver abzielen.</span><span class="sxs-lookup"><span data-stu-id="a4238-122">Deploy the application from a remote location by targeting the *Internet Information Services* (*IIS) Web Deploy Handler* on the destination server.</span></span>

<span data-ttu-id="a4238-123">Der Remote-Agent hat in diesem Fall zwei wichtige Einschränkungen:</span><span class="sxs-lookup"><span data-stu-id="a4238-123">The remote agent has two key limitations in this case:</span></span>

- <span data-ttu-id="a4238-124">Der Remote-Agent unterstützt nur die NTLM-Authentifizierung.</span><span class="sxs-lookup"><span data-stu-id="a4238-124">The remote agent supports only NTLM authentication.</span></span> <span data-ttu-id="a4238-125">Anders ausgedrückt: die Bereitstellung muss die builddienstidentität&#x2014;verwenden, Sie können die Identität eines anderen Kontos nicht annehmen.</span><span class="sxs-lookup"><span data-stu-id="a4238-125">In other words, the deployment must use the build service identity&#x2014;you can't impersonate another account.</span></span>

<span data-ttu-id="a4238-127">Durch diese beiden Einschränkungen wird der Remote-Agent-Ansatz für eine automatisierte TeamBuild-Bereitstellung unerwünscht.</span><span class="sxs-lookup"><span data-stu-id="a4238-127">Together, these two limitations make the remote agent approach undesirable for an automated Team Build deployment.</span></span> <span data-ttu-id="a4238-128">Um diese Vorgehensweise verwenden zu können, müssen Sie das Builddienstkonto als Administrator auf jedem Zielweb Server einrichten.</span><span class="sxs-lookup"><span data-stu-id="a4238-128">To use this approach, you'd need to make the build service account an administrator on any target web servers.</span></span>

<span data-ttu-id="a4238-129">Im Gegensatz dazu bietet der Web deploy handleransatz verschiedene Vorteile:</span><span class="sxs-lookup"><span data-stu-id="a4238-129">In contrast, the Web Deploy Handler approach offers various advantages:</span></span>

- <span data-ttu-id="a4238-130">Der Web deploy Handler unterstützt die Standard Authentifizierung über HTTPS, sodass Sie die Anmelde Informationen eines alternativen Kontos an das IIS-Webbereitstellungs Tool (Web deploy) übergeben können.</span><span class="sxs-lookup"><span data-stu-id="a4238-130">The Web Deploy Handler supports basic authentication over HTTPS, which allows you to pass the credentials of an alternative account to the IIS Web Deployment Tool (Web Deploy).</span></span>
- <span data-ttu-id="a4238-131">Sie können Zielweb Server so konfigurieren, dass Benutzer, die nicht Administratoren sind, Inhalte für bestimmte IIS-Websites mithilfe des Web deploy-Handlers bereitstellen können.</span><span class="sxs-lookup"><span data-stu-id="a4238-131">You can configure target web servers to allow non-administrator users to deploy content to specific IIS websites using the Web Deploy Handler.</span></span>

3. <span data-ttu-id="a4238-136">Rufen Sie Web deploy auf, und verwenden Sie für die Bereitstellung die Anmelde Informationen des Domänen Kontos, das Sie erstellt haben, um die Web deploy Handler als Ziel zu verwenden.</span><span class="sxs-lookup"><span data-stu-id="a4238-136">Invoke Web Deploy and target the Web Deploy Handler, using basic authentication and supplying the credentials of the domain account you created, to perform the deployment.</span></span>

<span data-ttu-id="a4238-137">In der Beispiellösung [Contact Manager](../web-deployment-in-the-enterprise/the-contact-manager-solution.md) geben Sie den Authentifizierungstyp (Basic oder NTLM), die Web deploy Anmelde Informationen und die Endpunkt Adresse (Remote-Agent oder Web deploy Handler) in der Umgebungs spezifischen Projektdatei an.</span><span class="sxs-lookup"><span data-stu-id="a4238-137">In the [Contact Manager](../web-deployment-in-the-enterprise/the-contact-manager-solution.md) sample solution, you specify the authentication type (basic or NTLM), the Web Deploy credentials, and the endpoint address (remote agent or Web Deploy Handler) in the environment-specific project file.</span></span> <span data-ttu-id="a4238-138">Diese Werte werden verwendet, um einen Web deploy Befehl zu formulieren und auszuführen, wenn die Projektdatei ausgeführt wird.</span><span class="sxs-lookup"><span data-stu-id="a4238-138">These values are used to formulate and run a Web Deploy command when the project file is executed.</span></span> <span data-ttu-id="a4238-139">Weitere Informationen finden Sie unter Bereitstellen von [Webpaketen](../web-deployment-in-the-enterprise/deploying-web-packages.md).</span><span class="sxs-lookup"><span data-stu-id="a4238-139">For more information, see [Deploying Web Packages](../web-deployment-in-the-enterprise/deploying-web-packages.md).</span></span>

<span data-ttu-id="a4238-140">Weitere Informationen zum Konfigurieren des Web deploy Handlers, einschließlich der Vorgehensweise zum Konfigurieren von Berechtigungen, finden Sie unter [Konfigurieren eines Webservers für die Web deploy Veröffentlichung (Web deploy Handler)](../configuring-server-environments-for-web-deployment/configuring-a-web-server-for-web-deploy-publishing-web-deploy-handler.md).</span><span class="sxs-lookup"><span data-stu-id="a4238-140">For more information on configuring the Web Deploy Handler, including how to configure permissions, see [Configuring a Web Server for Web Deploy Publishing (Web Deploy Handler)](../configuring-server-environments-for-web-deployment/configuring-a-web-server-for-web-deploy-publishing-web-deploy-handler.md).</span></span> <span data-ttu-id="a4238-141">Weitere Informationen zum Konfigurieren des Remote-Agents finden Sie unter [Konfigurieren eines Webservers für die Web deploy Veröffentlichung (Remote-Agent)](../configuring-server-environments-for-web-deployment/configuring-a-web-server-for-web-deploy-publishing-remote-agent.md).</span><span class="sxs-lookup"><span data-stu-id="a4238-141">For more information on configuring the remote agent, see [Configuring a Web Server for Web Deploy Publishing (Remote Agent)](../configuring-server-environments-for-web-deployment/configuring-a-web-server-for-web-deploy-publishing-remote-agent.md).</span></span>
