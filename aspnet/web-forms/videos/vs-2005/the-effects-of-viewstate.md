# <a name="the-effects-of-viewstate"></a>Die Auswirkungen von ViewState

von Eric Lee

Das ASP.NET-Framework stellt den ViewState-Mechanismus bereit, um den Anwendungs Zustand automatisch beizubehalten, indem Daten vorwärts und rückwärts zwischen Browser Anforderungen ausgeführt werden. In diesem Video sehen Sie, wie einige ASP.NET-Steuerelemente, wie z. b. Dropdown List und TextBox, den Status auf der Grundlage von Post Back Formular Werten und nicht ViewState behalten, sodass der intelligente Entwickler unnötige ViewState-Elemente deaktivieren kann. Mithilfe von Visual Studio Team System können wir sehen, wie dies die Größe der Anforderungs-und Antwortdaten drastisch reduziert und die Leistung und Skalierbarkeit der Anwendung verbessert.
