Stattdessen sind sie nur Teil des Rendervorgangs des Elements und des Inhalts.</span><span class="sxs-lookup"><span data-stu-id="dbbce-251">Tag Helpers don't own the element they're associated with; they simply participate in the rendering of the element and content.</span></span> <span data-ttu-id="dbbce-252">[Webserversteuerelemente](https://msdn.microsoft.com/library/7698y1f0.aspx) für ASP.NET werden auf einer Seite deklariert und aufgerufen.</span><span class="sxs-lookup"><span data-stu-id="dbbce-252">ASP.NET [Web Server controls](https://msdn.microsoft.com/library/7698y1f0.aspx) are declared and invoked on a page.</span></span>

* <span data-ttu-id="dbbce-258">Mehrere Taghilfsprogramme können gleichzeitig auf dasselbe Element wirken (weitere Informationen finden Sie unter [Avoiding Tag Helper conflicts (Vermeiden von Konflikten mit Taghilfsprogrammen)](xref:mvc/views/tag-helpers/authoring#avoid-tag-helper-conflicts)). Sie können hingegen in der Regel keine Webserversteuerelemente erstellen.</span><span class="sxs-lookup"><span data-stu-id="dbbce-258">Multiple Tag Helpers can act on the same element (see [Avoiding Tag Helper conflicts](xref:mvc/views/tag-helpers/authoring#avoid-tag-helper-conflicts) ) while you typically can't compose Web Server controls.</span></span>

* <span data-ttu-id="dbbce-259">Taghilfsprogramme können das Tag und den Inhalt von HTML-Elementen verändern, dem sie zugeordnet sind. Ansonsten nehmen Sie keine Änderungen an der Seite vor.</span><span class="sxs-lookup"><span data-stu-id="dbbce-259">Tag Helpers can modify the tag and content of HTML elements that they're scoped to, but don't directly modify anything else on a page.</span></span> <span data-ttu-id="dbbce-260">Der Funktionsbereich von Webserversteuerelementen ist weniger spezifisch. * <span data-ttu-id="dbbce-261">Webserversteuerelemente verwenden Typkonverter, um Zeichenfolgen in Objekte zu konvertieren.</span><span class="sxs-lookup"><span data-stu-id="dbbce-261">Web Server controls use type converters to convert strings into objects.</span></span> <span data-ttu-id="dbbce-262">Mit Taghilfsprogrammen arbeiten Sie auf native Weise in C#, weshalb Sie keine Typkonvertierung durchführen müssen.</span><span class="sxs-lookup"><span data-stu-id="dbbce-262">With Tag Helpers, you work natively in C#, so you don't need to do type conversion.</span></span>

* <span data-ttu-id="dbbce-263">Webserversteuerelemente verwenden [System.ComponentModel](/dotnet/api/system.componentmodel), um das Verhalten von Komponenten und Steuerelementen zur Laufzeit und Entwurfszeit zu implementieren.</span><span class="sxs-lookup"><span data-stu-id="dbbce-263">Web Server controls use [System.ComponentModel](/dotnet/api/system.componentmodel) to implement the run-time and design-time behavior of components and controls.</span></span> <span data-ttu-id="dbbce-264">`System.ComponentModel` enthält die Basisklassen und Schnittstellen zum Implementieren von Attributen und Typkonvertern, die Datenquellen binden und Komponenten lizenzieren.</span><span class="sxs-lookup"><span data-stu-id="dbbce-264">`System.ComponentModel` includes the base classes and interfaces for implementing attributes and type converters, binding to data sources, and licensing components.</span></span> <span data-ttu-id="dbbce-265">Im Gegensatz dazu stehen Taghilfsprogramme, die in der Regel von `TagHelper` abgeleitet sind. Die `TagHelper`-Basisklasse stellt nur zwei Methoden zur Verfügung: `Process` und `ProcessAsync`.</span><span class="sxs-lookup"><span data-stu-id="dbbce-265">Contrast that to Tag Helpers, which typically derive from `TagHelper`, and the `TagHelper` base class exposes only two methods, `Process` and `ProcessAsync`.</span></span>
